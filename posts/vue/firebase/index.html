      Introduction

      Firebase provides a SDK to easily integrate the auth service into our web app.

      The user credentials are collected using the web app. The credentials are sent to firebase via SDK. Firebase takes care of storing the user information on registration and authentication.

      Goals

      1. We want to implement an vue app with authentication.
        1. User should be able to view only some sections our web app without authentication.
        2. If an unauthenticated user tries to view app sections which require authentication he will be redirected to the login page.

      Design

      We will setup three routes to component mapping

      /         => <Dashboard/> or <Landing/>
/login    => <Login/>
/register => <Register/>

      Url path / will render </Dashboard> or <Landing/> depending on user authentication state. Without authentication / will render <Landing/> component. With authentication / will render <Dashboard/> component.

      Only authenticated user can access<Dashboard/>

      import { createStore } from 'vuex'

// Create a new store instance.
const store = createStore({
	state() {
		return {
			user: null,
			authState: 'Unknown' // Unknown, Authenticating, Authenticated
		}
	},
	getters: {
		user: () => state.user,
		isAuthenticated: () => state.authState === 'Authenticated' && !!state.user,
		isAuthenticating: () => !!state.authState === 'Authenticating',
		authState: () => state.authState,
	},
	mutations: {
		SET_USER(state, { user }) {
			state.user = user;
		},
		SET_AUTH_STATE(state, { authState }) {
			state.authState = authState;
		}
	},
	actions: {
		setUser({ commit }, { user }) {
			commit('SET_USER', { user })
		},
		setAuthState({ commit }, { authState }) {
			commit('SET_AUTH_STATE', { authState })
		}
	}
})

export default store;

      import { createRouter, createWebHistory } from 'vue-router';
import Dashboard from './components/Dashboard.vue';
import Login from './components/Login.vue';
import Register from './components/Register.vue';
import Landing from './components/Landing.vue';
import store from './store';

const router = createRouter({
	history: createWebHistory(),
	routes: [
		{
			path: '/',
			name: 'Main',
			component: () => {
				return store.state.authState === 'Authenticated' ? Dashboard : Landing;
			}
		},
		{
			path: '/login',
			name: 'Login',
			component: Login,
		},
		{
			path: '/register',
			name: 'Register',
			component: Register,
		},
	]
});

      Here we can directly access authState from store to check if the user is already authenticated and render appropriate component.

      We have created four vue components in src/components directory. The mapping from url path to component is in routes.js. firebase.js contains the firebase related code. Finally we manage minimal user state in store.js.

      User can try to visit either / /login, /register or any other path.

      Unauthenticated user can visit first 3 paths and the routes.js will render the mapped component. If he tries to visit any other path he will be redirected to /login.

      When a authenticated user tries to visit / the web app will render Dashboard component.If he tries to visit /login or /register path he will be redirected to / and lang in Dashboard.
